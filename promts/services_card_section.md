# Техническое задания на ветстку секций  

Необходимо сверстать несколько секций, по предоставленному техническому заданию. 

## Основные положения

 - При верстке используй все современные технологии но только те которые имеют повсеместную поддержку в разных браузерах
 - Для стилизации scss 
 - Стили для каждой секции должны лежать в отдельном файле. Для каждой секции файл будет указан в описании ниже.
 - В начале каждого файла со стилями должно быть подключение файла с переменными, реализованное следующим образом: @use '../vars' as *;.
 - При создании стилей используй переменные которые описанны в файле public\scss\_vars.scss. Не используй другие имена переменных! Используй только те которые есть в указанном файле.
 - Каждая секция должна быть обернута в семантический тег section с осмысленным классом и Id. Внутри каждой секции должен быть контейнер описанный следующим образом <div class="container"> </div>. Всю разметку секции делай внутри данного контейнера.
 - Без прямой необходимости или прямых указаний описанных в этом ТЗ не стилизуй секции и вложенный контейнер, стили для них уже прописанны. 
 - Без указаний в тексте задания не делай эффектов наведения для элементов.
 - Если по ходу выполнения работ нужно создать кнопку по умолчанию делай ее при помощи тега a с прописанным классом button. Стили для кнопки уже прописанны.
 - Созданные в ходе работы секции помещай в файл resources\views\index.blade.php перед закрывающим тегом @endsection
 

## Адаптация под устройства (общие требования)

- все секции должны быть адапрированы для мобильных устройств
- при написании стилей адаптации стариайся их минимизировать и не преписывать лишнее
- Мобильное устройство начинается от ширины экрана 520px
- Планшеты начинается от ширины экрана 820px
- Прими во внимание что для слайдеров у которых количество отображаемых элементов на десктоп версии > 1 нужно прописать брекпоинты при инициализации.
- Ниже в задании для каждой секции могут быть описанны требования для ее мобильной адаптации, учитывай эти требования.  

 
 ## Описание секций

 ### Секция - Карточки с цифрами

Секцию необходимо реализовать в файле компонента resources\views\components\services\number-card.blade.php

Стили поместить в файл public\scss\services\_number-card.scss

В компонент передаются следующие параметры 
- Заголовок
- Массив со заголовками

Внутри контейнера секции должен быть заголовок h2 в который выводится  параметр 'Заголовок'. 

Ниже должна быть обертка внутри которой расположенны карточки которые выводятся в цикле по переданному параметру 'Массив со заголовками'. Карточки размещаются по 3 в строку. 

**Описание структуры карточки:**

- Каждая карточка имеет обводку толщиной в 1 пиксель и цветом  $main_color. 
- У карточки внутренний отступ 10 пикселей. 
- В левой части карточки выводится ее индекс в формате '1.', '2.', '3.' и т. д. Индекс берется из встроенного параметра в цикл шаблонизатора @foreach. Шрифт жирный 24px, цвет $gray_border_color.
- В правой части выводится заголовок h3 Шрифт жирный 22px, цвет $black_color.

### Секция - Шаги

Секцию необходимо реализовать в файле компонента resources\views\components\services\steps.blade.php

Стили поместить в файл public\scss\services\_steps.scss

В компонент передаются следующие параметры 
- Заголовок
- Массив с заголовками.

Внутри контейнера секции должен быть заголовок h2 в который выводится  параметр 'title'. 

Ниже должна быть обертка внутри которой расположенны 2 столбца:
- В левом столбце распологается блок с описанием шагов
- В правом стобце фотография коткорую необходимо взять из файла public\img\steps_1.webp

**Описание блока с описанием шагов:**

Блок шагов должен представлять из себя вертикальную линию с расположенными в кругах индексами шагов на ней. Названия шагов расположенны с права от круга с индексом. Цвет линии и кругов $main_color.

Шаги выводятся в цикле по переданному параметру 'Массив с заголовками'. Индексы в формате '1.', '2.', '3.' и т. д. Наименования шагов выводятся в параграфах со следующими параметрами шрифта: жирный 22px, цвет $black_color.
