# Техническое задание на разработку бэкэнда

## Создание структуры БД

Для каждой перечисленной в этом разделе сущьности нужно создать:
- Модель
- Миграцию
- Сидер (Сидер создается пустой, к нему подключается фасад DB и Str)
- Контроллер (если в описании сущьности будет указание контроллер не создаем)

При наличии в описании полей сущьности поля Слаг/slug нужно создать мутатор который делает это поле из поля заголовка или имени.

Если в описании сущьности прошу создать роуты то добавляем их в файл routes\asmi_all.php и подключаем соответствующий контроллер.

Если в описании прошу создать views то для сущьности создается отдельная папка и в ней 2 файла:
- index - для отображения общего списка
- page - для отображение элемента сущьности

Шаблон для этих файлов следующий:

```php
@extends('layouts.all')

@section('main')
    <h1></h1>
    
@endsection

```

### Сущьность - Специалист

**Необходимые поля:**
- Фио (строка)
- Фото (строка, 550 символов)
- Слаг (строка)
- Должность (строка, 150 символов)
- Описание (текст, nullable)
- Порядок вывода (Целое число, по умолчанию 0)
- Сертификаты (json)

В котнтроллере необходимо создать метод для вывода страницы со всеми специалистами а так же для вывода отдельной страницы каждого специалиста. Добавь роуты для этой сущьности. Так же добавь views.


### Сущьность - Цены

**Необходимые поля:**
- Код услуги (строка, 100 символов)
- Наименование услуги (строка, 400 символов)
- Цена (строка, 50 символов)
- Единица измерения (строка, 50 символов, nullable)
- Раздел (текст, nullable)
- Порядок вывода (Целое число, по умолчанию 0)

В котнтроллере необходимо создать метод для вывода страницы со всеми ценами. Страницу под вывод каждой цены не создавай. Добавь роут для этой сущьности, с выводом общей старницы цен. Так же добавь views.

### Сущьность - Баннер

**Необходимые поля:**
- Название баннера (строка, 100 символов)
- Заголовок (строка, nullable)
- Порядок вывода (Целое число, по умолчанию 0)
- Изображение (строка, 500 символов)
 
Для данной сущьности НЕ СОЗДАЕМ: контроллер, роуты, views.

### Сущьность - Преимущество

**Необходимые поля:**
- Заголовок (строка, 100 символов)
- Описание (строка)
- Изображение (строка, 500 символов)
- Порядок вывода (Целое число, по умолчанию 0)

Для данной сущьности НЕ СОЗДАЕМ: контроллер, роуты, views.
